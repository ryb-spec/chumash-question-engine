# BHH Chumash AI pilot one-page brief

## What this app is

This is an early Chumash practice prototype designed to help students build independent skills through short, adaptive practice questions.

## What it currently does

- Serves Chumash practice in a supported Learn Mode.
- Uses a trusted active runtime scope rather than open-ended generated content.
- Tracks teacher-pilot evidence through logs, review exports, and manual observations.
- Supports teacher monitoring so confusing or weak questions can be found quickly.

## What it does not yet do

- It is not ready for schoolwide rollout.
- It does not cover all Chumash skills.
- It does not activate Perek 4.
- It does not automatically promote pilot items to reviewed-bank or runtime status.
- It does not replace teacher review.

## What skills the pilot focuses on

The immediate pilot focuses on whether current Perek 3 practice items are clear, fair, and instructionally useful, especially for basic word recognition and prompt clarity.

## Why this small pilot matters

The project now has strong process and validation infrastructure. The next bottleneck is real student and teacher evidence. A small fresh pilot is the responsible way to learn what actually helps students and what still needs repair.

## How content accuracy is protected

- The app uses local source data and reviewed runtime scope.
- Validators and release checks guard against unsafe scope expansion.
- Teacher observations are recorded before broader claims are made.
- Pilot evidence does not automatically approve content.

## How teacher feedback is used

Teacher feedback is used to classify each observed concern, identify repeated issues, and decide whether an item should stay limited, be revised, need source follow-up, or be rejected for broader use.

## What success looks like after one pilot

Success means 3-5 students complete short Learn Mode sessions, the teacher monitor is open, real observations are recorded, and the team has a small list of concrete issues or confidence signals.

## What is intentionally not being expanded yet

- No Perek 4 runtime activation.
- No reviewed-bank promotion.
- No broad new standards layer.
- No schoolwide rollout.
- No auth/database build.
- No large refactor.

## Next responsible step

Run the fresh Perek 3 pilot with 3-5 students in Learn Mode only, record observations with the rubric, and fix only the top repeated issues before considering any further expansion.
