# Bereishis Perek 1 First-Batch Answer-Key Planning Policy

This policy is answer-key planning only. It does not create answer keys for individual rows.

## vocabulary_meaning

- Later answer key must use only reviewed meaning.
- If token includes article/prefix, answer-key review must decide: base meaning only, article-inclusive meaning, or context-sensitive acceptable variants.
- Do not create actual answers in this task.

## basic_noun_recognition

- Later answer key must identify only simple category labels: noun, person, place, thing, object.
- No advanced grammar labels.
- No construct/suffix/adjective claims without separate review.
- Do not create actual answers in this task.

## direct_object_marker_recognition

- Later answer key must explain function, not translation.
- Must not say `את = the`.
- Must not say `את = with`.
- Must not treat `את` as a normal vocabulary word.
- Do not create actual answers in this task.

## shoresh_identification

- Later answer key must provide target shoresh only after review.
- For `הבדיל`, target shoresh is `בדל`.
- Must distinguish surface form from shoresh.
- Do not create actual answers in this task.

## Deferred Family

`basic_verb_form_recognition` remains excluded.
